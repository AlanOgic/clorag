     GraphRAG Implementation Plan for CLORAG

     Overview

     Add a knowledge graph layer using Neo4j to augment the existing hybrid 
     vector search (Voyage AI + BM25). The graph will store entities
     (cameras, products, protocols, issues) and their relationships
     extracted from documentation and support cases.

     Technology Choices

     - Graph Database: Neo4j (Docker container, v5.x community)
     - Python Driver: neo4j>=6.0.0 (async API)
     - LangChain Integration: langchain-neo4j>=0.6.0 for graph utilities
     - Entity Extraction: Claude Haiku 4.5 (reuse existing pattern from 
     camera_extractor.py)
     - Integration Mode: Hybrid - vector search finds chunks, graph enriches
      context

     Neo4j Schema

     Nodes

     (:Camera {name, manufacturer, doc_url, created_at})
     (:Product {name, type, doc_url})  # RIO, RCP, CI0, VP4, Live Composer
     (:Protocol {name, type})          # VISCA, LANC, Pelco, Canon XC, etc.
     (:Port {name, type})              # Ethernet, RS-485, HDMI, USB, etc.
     (:Control {name, description})    # Iris, Focus, Zoom, etc.
     (:Issue {description, symptoms})
     (:Solution {description, steps})
     (:Firmware {version, product, release_date})
     (:Chunk {id, collection, title})  # Reference to Qdrant chunks

     Relationships

     (Camera)-[:COMPATIBLE_WITH]->(Product)
     (Camera)-[:USES_PROTOCOL]->(Protocol)
     (Camera)-[:HAS_PORT]->(Port)
     (Camera)-[:SUPPORTS_CONTROL]->(Control)
     (Product)-[:SUPPORTS_PROTOCOL]->(Protocol)
     (Product)-[:HAS_PORT]->(Port)
     (Issue)-[:AFFECTS]->(Product|Camera)
     (Issue)-[:RESOLVED_BY]->(Solution)
     (Solution)-[:MENTIONED_IN]->(Chunk)
     (Firmware)-[:FOR_PRODUCT]->(Product)
     (Firmware)-[:FIXES]->(Issue)
     (Chunk)-[:MENTIONS]->(Camera|Product|Protocol)

     Files to Create

     1. src/clorag/core/graph_store.py

     Neo4j async client wrapper with:
     - Connection pool management (singleton pattern)
     - CRUD operations for all node types
     - Graph traversal queries (multi-hop)
     - Relationship management
     - Batch upsert for entity population

     2. src/clorag/core/entity_extractor.py

     LLM-based entity extraction using Claude Haiku:
     - Extract entities from text chunks (cameras, products, protocols, 
     issues, solutions)
     - Extract relationships between entities
     - Batch processing with concurrency control
     - Structured output using Pydantic models

     3. src/clorag/graph/schema.py

     Pydantic models for graph entities:
     - GraphCamera, GraphProduct, GraphProtocol, etc.
     - Relationship models with source/target typing
     - EntityExtractionResult for LLM output parsing

     4. src/clorag/graph/enrichment.py

     Context enrichment service:
     - Given chunk IDs from vector search, find related entities
     - Traverse graph to find connected information
     - Format graph context for Claude synthesis

     4. src/clorag/ingestion/docusaurus.py

     Add optional graph population:
     - After embedding chunks, extract entities

     Search Flow Integration

     Query → Vector Search (Qdrant) → Top-K Chunks
                                          ↓
                                Graph Enrichment (Neo4j)
                                          ↓
                         Retrieve related entities & relationships
                                          ↓
                                Build Enhanced Context
                                          ↓
                             Claude Synthesis (with graph info)

     Implementation Phases

     Phase 1: Core Infrastructure

     1. Add neo4j and langchain-neo4j dependencies
     2. Create graph_store.py with connection handling
     3. Create schema.py with Pydantic models
     4. Add Neo4j to docker-compose.yml
     5. Update config.py with Neo4j settings

     Phase 2: Entity Extraction

     1. Create entity_extractor.py with Claude Haiku
     2. Design extraction prompts for each entity type
     3. Add batch processing with rate limiting
     4. Test extraction on sample documents

     Phase 3: Graph Population

     1. Create CLI command: uv run populate-graph
     2. Process existing Qdrant chunks
     3. Create nodes and relationships
     4. Add to ingestion pipelines (optional flag)

     Phase 4: Search Integration

     1. Create enrichment.py for context enhancement
     2. Modify app.py search endpoint
     3. Update synthesis prompt to use graph context
     4. Add graph stats to admin dashboard
