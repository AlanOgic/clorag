API Reference

This page documents the public API endpoints available without authentication. For administrative endpoints, refer to the admin documentation.

Search API

POST /api/search

Perform RAG search with Claude-powered synthesis.

{
  "query": "How to configure RIO IP address?",
  "source": "both",  // "docs", "gmail", or "both"
  "limit": 10,
  "session_id": null  // Optional: UUID from previous response for follow-ups
}

Response includes session_id to use in subsequent requests for follow-up conversations.

POST /api/search/stream

Streaming version with Server-Sent Events (SSE). First event sends {"type": "session", "session_id": "..."}. Subsequent events stream the AI-generated answer in real time.

Camera API

List & Search

GET /api/cameras

List all cameras. Supports filters: manufacturer, device_type, port, protocol.

GET /api/cameras/search?q={query}

Search cameras by name or manufacturer using full-text search with BM25 ranking.

GET /api/cameras/{id}

Get a single camera by ID.

Comparison & Related

POST /api/cameras/compare

Compare multiple cameras side-by-side. Request body: {"camera_ids": [1, 2, 3]} (max 5).

GET /api/cameras/{id}/related

Get cameras similar to the specified camera. Query param: limit (default: 5).

Export

GET /api/cameras/export.csv

Export cameras as CSV. Optional filters: manufacturer, device_type.

OpenAI-Compatible API

CLORAG exposes an OpenAI-compatible API that allows any OpenAI SDK client to query the RAG system. Requires an API key for authentication.

Authentication

All requests require a Bearer token:

Authorization: Bearer your_api_key_here
POST /v1/chat/completions

OpenAI-compatible chat completions. The last user message becomes the RAG search query. Prior user/assistant pairs are forwarded as conversation history. System messages are accepted but ignored.

{
  "model": "clorag",
  "messages": [
    {"role": "user", "content": "How to configure RIO IP address?"}
  ],
  "stream": false
}

Response follows the standard OpenAI ChatCompletion format with sources appended as markdown links:

{
  "id": "chatcmpl-abc123...",
  "object": "chat.completion",
  "created": 1714000000,
  "model": "clorag",
  "choices": [{
    "index": 0,
    "message": {
      "role": "assistant",
      "content": "Here is how to configure...\n\n---\n**Sources:**\n- [RIO Guide](https://...)"
    },
    "finish_reason": "stop"
  }],
  "usage": {"prompt_tokens": 0, "completion_tokens": 0, "total_tokens": 0}
}

Set "stream": true for SSE streaming in OpenAI chunk format (chat.completion.chunk).

GET /v1/models

List available models. Returns the clorag model.

Python SDK Example

from openai import OpenAI

client = OpenAI(
    base_url="https://your-clorag-instance",
    api_key="your_api_key",
)

response = client.chat.completions.create(
    model="clorag",
    messages=[{"role": "user", "content": "How to configure RIO?"}],
)
print(response.choices[0].message.content)
Accepted but Ignored Parameters

temperature, top_p, max_tokens, max_completion_tokens, frequency_penalty, presence_penalty, n, and user are accepted without error but have no effect — CLORAG controls its own RAG synthesis parameters.

Feedback API

POST /api/feedback

Submit thumbs up/down feedback for a search result. Upserts per search_id (one vote per search, can be changed).

{
  "search_id": "uuid-from-search-response",
  "rating": "up",       // "up" or "down"
  "comment": "optional improvement suggestion"
}

Messages API

GET /api/messages

Get active (non-expired) messages for display on the index page. Returns announcements with type (info, warning, feature, fix), title, content, and expiration.
