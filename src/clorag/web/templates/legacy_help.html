How to Use Legacy Docs

Search, manage, and keep the legacy index up to date

What is this?

Legacy Docs is a standalone search engine for the existing support.cyanview.com documentation. It uses the same AI-powered RAG technology as the main Cyanview search, but operates on its own separate index.

Use it to search the current support site content, find pages that need improvement, and keep the index in sync as you update pages.

This system is completely independent from the main CLORAG search engine. Nothing you do here affects the main search at /.

Typical Workflow

1
Search the legacy docs to find content that needs updating
2
Click the source link to go directly to the page on support.cyanview.com
3
Edit the page on support.cyanview.com
4
Re-ingest the page — go to Manage, paste the URL, click "Ingest". The index now reflects your changes.
5
Search again to verify the updated content appears correctly

Manage Actions

Scan Sitemap

Compares the live sitemap of support.cyanview.com against the index. Shows any new pages not yet indexed. You can select and ingest them in one click.

Use this when you've created new pages on the site.

Ingest Single Page

Paste the full URL of a page you just modified (e.g. https://support.cyanview.com/docs/Configuration/REMI). It deletes the old chunks for that page and re-indexes the fresh content.

Use this when you've modified an existing page.

Full Re-ingest

Deletes everything in the index and re-crawls the entire site from scratch. Takes a few minutes. Use this when many pages have changed or as a periodic refresh.

Rate limited to 2 per hour. The crawl uses Jina Reader for content extraction — for large batches this can take 3-5 minutes.

Technical Details

Collection: docusaurus_docs_legacy (Qdrant) — separate from docusaurus_docs

Embeddings: Voyage AI voyage-context-3 (1024-dim contextualized)

Sparse vectors: BM25 for keyword matching

Reranking: Voyage rerank-2.5 (when enabled in settings)

Synthesis: Claude Sonnet 4.6

Source site: https://support.cyanview.com
