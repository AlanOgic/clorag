MCP Server

Overview

The CLORAG MCP (Model Context Protocol) server exposes the full RAG knowledge base to AI assistants like Claude Desktop, Claude Code, and any MCP-compatible client. It provides 33 tools, 8 resources, 5 resource templates, and 6 prompt templates for searching, managing, and ingesting Cyanview support data.

MCP Protocol

MCP is an open standard for connecting AI assistants to external data sources. The server uses FastMCP from the mcp SDK v1.22.0.

Capabilities Summary

TypeCountDescription
Tools33Actions: search, CRUD, ingestion, prompts, maintenance
Resources8Static data: stats, products, categories, prompts, search quality
Templates5Parameterized: camera, document, case, prompt by key/versions
Prompts6Guided workflows: troubleshoot, compare, integrate

Setup

Claude Desktop (stdio)

Add to your claude_desktop_config.json:

{
  "mcpServers": {
    "clorag": {
      "command": "uv",
      "args": ["run", "clorag-mcp"],
      "cwd": "/path/to/clorag"
    }
  }
}

Claude Code

Add to your .mcp.json in the project root:

{
  "mcpServers": {
    "clorag": {
      "type": "stdio",
      "command": "uv",
      "args": ["run", "clorag-mcp"],
      "cwd": "/path/to/clorag"
    }
  }
}

Docker / Remote (HTTP)

# docker-compose.yml
services:
  clorag-mcp:
    build: .
    command: uv run clorag-mcp-http
    ports:
      - "8000:8080"
    environment:
      - MCP_HOST=0.0.0.0
      - MCP_PORT=8080

Search Tools

Hybrid RAG search across all three vector collections with RRF fusion and Voyage reranking.

Tool async search
Search all sources using hybrid RAG with reranking. Main entry point for queries.
query: str · source: "docs" | "cases" | "custom" | "both" · limit: 1-20
Tool async search_docs
Search official Docusaurus documentation only. Best for product features, how-to guides, and specs.
query: str · limit: 1-10
Tool async search_cases
Search past Gmail support cases. Best for similar issues, resolutions, and real-world examples.
query: str · limit: 1-10

Chunk Management Tools

Low-level access to Qdrant vector chunks. Read, edit, and search within specific collections with metadata filtering.

Collection Names

All chunk tools accept short names: "docs", "cases", "custom" (mapped to docusaurus_docs, gmail_cases, custom_docs).

Tool async list_chunks
Paginated listing of chunks. Optionally filter by metadata field (e.g. all chunks from one URL or thread).
collection: str · limit: 1-50 · offset: str? · field: str? · value: str?
Tool async get_chunk
Get full chunk details: text, metadata, chunk position within parent document.
collection: str · chunk_id: str
Tool async edit_chunk
Edit chunk text and/or metadata. Text changes trigger automatic re-embedding (dense + sparse). Marks chunk as human_edited to prevent overwrite during re-ingestion.
collection: str · chunk_id: str · text: str? · title: str? · subject: str?
Tool async delete_chunk
Permanently delete a chunk and its vectors from a collection. Irreversible.
collection: str · chunk_id: str
Tool async search_chunks
Hybrid RRF search within a single collection with optional metadata filter. Over-fetches 3x and reranks with Voyage rerank-2.5.
query: str · collection: str · limit: 1-20 · field: str? · value: str?

Ingestion & Maintenance Tools

Data pipeline operations. Most are long-running and consume API credits (Voyage embeddings, Anthropic Sonnet).

Long-running Operations

Ingestion tools can take minutes to hours. They use Anthropic and Voyage API credits. Use with care.

Tool async slow ingest_docs
Ingest Docusaurus documentation. Fetches pages via Jina Reader, extracts keywords (Sonnet), chunks, generates hybrid embeddings, stores in Qdrant.
fresh: bool (delete collection first) · extract_cameras: bool
Tool async slow ingest_curated
Ingest curated Gmail support threads. 7-step pipeline: Fetch, Filter RMA, Anonymize, Analyze (Sonnet), QC (Sonnet), Chunk & Embed, Store.
max_threads: int? · offset: int · min_confidence: float · fresh: bool · extract_cameras: bool · since_days: int?
Tool async import_custom_documents
Bulk import .txt, .md, .pdf files from a folder into the custom knowledge base.
folder: str (absolute path) · category: str · tags: str (comma-separated) · dry_run: bool
Tool async slow enrich_cameras
Enrich camera database with official model codes and manufacturer URLs via web search + Sonnet extraction.
manufacturer: str? · limit: int? · dry_run: bool · enrich_urls: bool
Tool async slow populate_graph
Extract entities from Qdrant chunks and populate the Neo4j knowledge graph. Requires NEO4J_PASSWORD.
collections: list[str]? · batch_size: int · max_chunks: int? · concurrency: int
Tool sync rebuild_fts_index
Rebuild the camera FTS5 full-text search index. Fast operation (seconds).
Tool async slow fix_rio_preview
Scan all collections for RIO terminology issues. Saves suggestions for admin review at /admin/terminology-fixes.
max_chunks: int?
Tool async slow fix_rio_apply
Apply approved RIO fixes. Groups by document, re-embeds all sibling chunks with contextualized embeddings.
Tool sync init_prompts_db
Initialize the prompt database with default LLM prompts. Skips existing prompts unless force=True.
force: bool (WARNING: resets customizations)

Prompt Management Tools

Edit and rollback LLM prompts used across the RAG pipeline. Changes are versioned for audit and rollback.

Tool sync update_prompt
Update a prompt's content or metadata. Content changes create a new version in history. Variables are auto-detected.
prompt_id: str (UUID) · content: str? · name: str? · description: str? · change_note: str?
Tool sync rollback_prompt
Rollback a prompt to a previous version. Creates a new version with the old content, preserving full history.
prompt_id: str (UUID) · version: int
Prompt Workflow

List prompts via clorag://prompts resource, get details via clorag://prompt/{key}, check history via clorag://prompt/{id}/versions, then use update_prompt or rollback_prompt tools.

Resources vs Tools

Resources are for reading data (clients can subscribe to updates). Tools are for actions (search, edit, ingest). Use resource templates when you know the exact ID; use tools when you need to search or filter.

Prompt Templates

Pre-built conversation starters that guide the AI assistant through structured workflows.

Prompt troubleshoot
Guided troubleshooting: searches docs + cases, identifies causes, provides step-by-step resolution.
issue: str (required) · product: str (optional)
Prompt compare_cameras
Side-by-side camera comparison for Cyanview compatibility. Searches DB, highlights ports/protocols.
camera_names: str (required)
Prompt find_camera
Find cameras matching a use case. Recommends Cyanview products and connection methods.
use_case: str (required) · requirements: str (optional)
Prompt explain_product
Detailed product explanation: variants, features, use cases, licensing, connections.
product: str (required, e.g. "RIO", "CI0")
Prompt integration_guide
Generate a full integration guide: hardware, network, step-by-step setup, pitfalls.
scenario: str (required)
Prompt ingest_status
Analyze knowledge base health and suggest which ingestion operations to run.

Architecture

Source Layout

src/clorag/mcp/
  server.py            # FastMCP server, lifespan, MCPServices
  resources.py         # 5 resources + 3 resource templates
  prompts.py           # 6 prompt templates
  tools/
    __init__.py        # Registration exports
    search.py          # search, search_docs, search_cases
    cameras.py         # search/get/list/compare/related cameras
    documents.py       # list/get documents, categories
    support.py         # search/get/list cases, raw thread, stats
    chunks.py          # list/get/edit/delete/search chunks
    ingestion.py       # ingest, enrich, populate, rebuild, fix
    prompts.py         # update_prompt, rollback_prompt

Service Initialization

MCPServices is initialized once in the lifespan context manager and shared across all tool/resource handlers. It holds:

  • embeddings / sparse_embeddings — Voyage AI clients
  • vectorstore — AsyncQdrantClient wrapper
  • reranker — Voyage rerank-2.5 cross-encoder
  • retriever — MultiSourceRetriever (combines all above)
  • camera_db / support_case_db — SQLite with connection pools
  • document_service — Custom document CRUD
