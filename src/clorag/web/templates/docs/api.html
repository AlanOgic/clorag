API Reference

Connection Guide

Base URL

Production: https://cyanview.cloud
Local Dev:  http://localhost:8080

Authentication Methods

1. Session Cookie (Recommended for Browser)

Login via the web form or API to receive a signed session cookie:

# Login
curl -X POST https://cyanview.cloud/api/admin/login \
  -H "Content-Type: application/json" \
  -d '{"password": "your_admin_password"}' \
  -c cookies.txt

# Use cookie for subsequent requests
curl https://cyanview.cloud/api/admin/search-stats \
  -b cookies.txt

2. X-Admin-Password Header (For Scripts)

Include the admin password in the header for programmatic access:

curl -X POST https://cyanview.cloud/api/admin/cameras \
  -H "Content-Type: application/json" \
  -H "X-Admin-Password: your_admin_password" \
  -d '{"name": "Canon C70", "manufacturer": "Canon"}'

Search API

POST /api/search

Perform RAG search with Claude-powered synthesis.

{
  "query": "How to configure RIO IP address?",
  "source": "both",  // "docs", "gmail", or "both"
  "limit": 10,
  "session_id": null  // Optional: UUID from previous response for follow-ups
}

Response includes session_id to use in subsequent requests for follow-up conversations.

POST /api/search/stream

Streaming version with Server-Sent Events (SSE). First event sends {"type": "session", "session_id": "..."}.

Camera API

Public Endpoints

GET /api/cameras

List all cameras. Supports filters: manufacturer, device_type, port, protocol.

GET /api/cameras/search?q={query}

Search cameras by name or manufacturer.

GET /api/cameras/{id}

Get a single camera by ID.

Comparison & Related

POST /api/cameras/compare

Compare multiple cameras side-by-side. Request body: {"camera_ids": [1, 2, 3]} (max 5).

GET /api/cameras/{id}/related

Get cameras similar to the specified camera. Query param: limit (default: 5).

Export/Import

GET /api/cameras/export.csv

Export cameras as CSV. Optional filters: manufacturer, device_type.

POST /api/admin/cameras/import

Import cameras from CSV file (admin only). Uses multipart/form-data with file field. Upserts based on name + manufacturer.

Admin Endpoints (Auth Required)

POST /api/admin/cameras

Create a new camera entry.

PUT /api/admin/cameras/{id}

Update an existing camera.

DELETE /api/admin/cameras/{id}

Delete a camera entry.

Knowledge Base API

GET /api/admin/knowledge

List all custom documents with optional filters: category, include_expired.

GET /api/admin/knowledge/categories

Get list of available document categories.

GET /api/admin/knowledge/{id}

Get a single custom document by ID.

POST /api/admin/knowledge

Create a new custom document from JSON. Chunks, embeds, and stores in Qdrant.

{
  "title": "RIO v2.5.0 Release Notes",
  "content": "Full document content here...",
  "category": "firmware",
  "tags": ["rio", "firmware", "v2.5.0"],
  "url_reference": "https://...",
  "expires_at": "2025-12-31",
  "notes": "Internal admin notes"
}
POST /api/admin/knowledge/upload

Upload a file (.txt, .md, .pdf) as a custom document. Uses multipart/form-data.

Form fields:
  file: [binary file data]    // Required: .txt, .md, or .pdf
  title: "Document Title"     // Optional: uses filename if empty
  category: "firmware"        // Optional: default "other"
  tags: "rio, v2.5.0"        // Optional: comma-separated
  url_reference: "https://..." // Optional
  notes: "Admin notes"        // Optional
PUT /api/admin/knowledge/{id}

Update a custom document. If content changes, re-embeds all chunks.

DELETE /api/admin/knowledge/{id}

Delete a custom document and all its chunks from Qdrant.

Chunk API

GET /api/admin/chunks

List chunks with pagination. Query params: collection, search, limit, offset.

GET /api/admin/chunks/{id}

Get a single chunk by ID with full content and metadata.

PUT /api/admin/chunks/{id}

Update chunk content or metadata. Triggers re-embedding if text changed. Sets human_modified=true.

{
  "text": "Updated chunk content...",
  "title": "Updated Title",
  "url": "https://..."
}
DELETE /api/admin/chunks/{id}

Delete a chunk from the vector store.

Support Cases API

GET /api/admin/support-cases

List support cases with pagination. Query params: category, product, limit, offset.

GET /api/admin/support-cases/stats

Get support case statistics: total count, breakdown by category, product, and resolution quality.

GET /api/admin/support-cases/search?q={query}

Search support cases using FTS5 full-text search. Query param: limit (default: 20).

GET /api/admin/support-cases/{id}

Get full support case details including problem/solution summaries, document, and metadata.

GET /api/admin/support-cases/{id}/raw-thread

Get the cleaned raw thread content for a case (quoted replies and signatures removed).

DELETE /api/admin/support-cases/{id}

Delete a support case from the database.

Analytics API

GET /api/admin/search-stats?days=30

Get search statistics.

GET /api/admin/search/{id}

Get stored search details including response and chunks.

Draft API

GET /api/admin/drafts/status

Get draft system status including pending count and last run time.

GET /api/admin/drafts/pending

List unanswered threads that need draft replies.

POST /api/admin/drafts/preview/{thread_id}

Generate AI draft preview without creating in Gmail.

POST /api/admin/drafts/create/{thread_id}

Create draft in Gmail for the specified thread.

POST /api/admin/drafts/run

Run the full draft pipeline for all pending threads.

Error Handling

StatusMeaningAction
200SuccessProcess response
401UnauthorizedCheck password/session
404Not FoundCheck resource ID
429Rate LimitedWait and retry (see Retry-After)
500Server ErrorCheck server logs
503Admin Not ConfiguredSet ADMIN_PASSWORD env var
Rate Limiting

API endpoints are rate-limited. Login attempts: 10/minute. Camera mutations: 5/minute. General API: no explicit limit but abuse may be blocked.
