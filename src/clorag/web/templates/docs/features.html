Draft Auto-Reply System

Overview

The draft system automatically generates email replies for unanswered support threads using RAG search and Claude AI synthesis.

Pipeline Flow

┌─────────────────────────────────────────────────────────────────────┐
│                    DRAFT CREATION PIPELINE                          │
└─────────────────────────────────────────────────────────────────────┘

┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│    GMAIL      │     │   ANALYZE     │     │    FILTER     │
│    FETCH      │────▶│   THREADS     │────▶│   PENDING     │
│  via API      │     │ ThreadAnalyzer│     │ no cyanview   │
└───────────────┘     └───────────────┘     └───────────────┘
                                                    │
        ┌───────────────────────────────────────────┘
        ▼
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│ CHECK DRAFTS  │     │  RAG SEARCH   │     │    CLAUDE     │
│  in Gmail     │────▶│ docs + cases  │────▶│   SYNTHESIS   │
│ skip existing │     │ hybrid query  │     │ draft content │
└───────────────┘     └───────────────┘     └───────────────┘
                                                    │
        ┌───────────────────────────────────────────┘
        ▼
┌───────────────┐
│ CREATE DRAFT  │
│  in Gmail     │
│ In-Reply-To   │
└───────────────┘

Key Components

ComponentDescription
GmailDraftServiceGmail API wrapper with compose scope for creating draft replies
DraftResponseGeneratorRAG search + Claude synthesis for contextual email content
DraftCreationPipelineOrchestrates the full process: fetch → analyze → filter → generate → create

Admin Interface

Access the draft management UI at /admin/drafts:

  • Pending Threads - List of unanswered threads with expandable message history
  • Thread Timeline - Visual timeline showing all messages with Cyanview badges
  • Gmail Links - Direct links to open threads in Gmail
  • Preview Draft - View AI-generated response before creating
  • Create Draft - Create the draft reply in Gmail
  • Run Pipeline - Batch process all pending threads

CLI Command

# Process all unanswered threads (max 10)
uv run draft-support

# Preview drafts without creating
uv run draft-support --preview

# Process specific thread
uv run draft-support --thread THREAD_ID

# Increase limit
uv run draft-support --max 20

Google Groups Handling

The system correctly handles messages arriving via Google Groups:

  • Parses X-Original-Sender header for actual customer email
  • Falls back to Reply-To header when available
  • Strips "via Group" suffix from display names
  • Correctly identifies Cyanview vs customer messages
OAuth Scope

Draft creation requires gmail.compose scope. If using existing OAuth token, delete token_draft.json to re-authenticate with new scopes.

Camera Database

Schema

The camera compatibility database stores structured information:

FieldTypeDescription
idINTEGERPrimary key
nameTEXTCamera model name
manufacturerTEXTManufacturer (Canon, Sony, etc.)
device_typeTEXTType (cinema, broadcast, PTZ, etc.)
portsJSONPhysical connection ports
protocolsJSONCommunication protocols
supported_controlsJSONControllable features
notesJSONAdditional notes
doc_urlTEXTDocumentation URL
manufacturer_urlTEXTManufacturer product page
sourceTEXTData source (documentation, support_case, manual)

Data Sources

Camera data is extracted from multiple sources:

  • Documentation - Automatic extraction during docs ingestion
  • Support Cases - Extraction from resolved support threads
  • Manual Entry - Admin interface for manual additions

Admin Interface

Access the camera manager at /admin/cameras:

  • Browse all cameras with filtering by manufacturer and type
  • Add new camera entries manually
  • Edit existing camera information
  • Delete outdated entries

Enrichment Command

# Enrich camera model codes from documentation
uv run enrich-cameras

This command scans documentation for camera model codes and enriches existing database entries with additional information.

Analytics System

Data Collected

Each search logs the following data:

  • Query text and source filter
  • Response time (milliseconds)
  • Number of results
  • Full LLM response text
  • Retrieved chunks (for replay without re-query)
  • Timestamp

Statistics Available

  • Total searches (with time range)
  • Average response time
  • Searches by source (docs/gmail/both)
  • Daily search volume chart
  • Popular queries (ranked)
  • Recent searches (clickable to view details)

Admin Interface

Access the analytics dashboard at /admin/analytics:

  • Time-filtered statistics (7d, 30d, all time)
  • Visual charts for search volume trends
  • Click any search to view full details including response and sources
