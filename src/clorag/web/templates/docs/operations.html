Operations

Deployment

Docker Deployment

# Build and deploy
rsync -avz --exclude '.venv' --exclude 'data' . root@server:/opt/clorag/
ssh root@server "cd /opt/clorag && docker compose build && docker compose up -d"

Web UI is accessible at https://cyanview.cloud/ (reverse proxy to port 8085).

CLI Commands

Server

# Run web server (FastAPI on port 8080)
uv run rag-web

# Run CLI agent (interactive or single query)
uv run clorag
uv run clorag "How do I configure the RIO IP address?"

Ingestion

# Ingest documentation from Docusaurus
uv run ingest-docs

# Ingest Gmail threads (curated with LLM analysis)
uv run ingest-curated --max-threads 300

# Incremental ingestion (skip first N threads)
uv run ingest-curated --offset 300 --max-threads 300

# Enrich camera model codes from documentation
uv run enrich-cameras

# Rebuild camera FTS5 search index
uv run rebuild-fts

# Check FTS index status
uv run rebuild-fts --check

Knowledge Graph

# Populate Neo4j knowledge graph from Qdrant chunks
uv run populate-graph
uv run populate-graph --collections docusaurus_docs gmail_cases
uv run populate-graph --max-chunks 100  # Limit for testing

Bulk Import

# Bulk import custom documents from a folder
uv run import-docs ./folder --category pre_sales --tags "presales,integration"
uv run import-docs ./folder --dry-run  # Preview without importing

Drafts

# Draft auto-reply for unanswered threads
uv run draft-support
uv run draft-support --preview  # Preview without creating

Development

# Linting and type checking
uv run ruff check src/
uv run mypy src/clorag --strict

# Tests
uv run pytest
uv run pytest tests/test_file.py::test_name -v  # Single test

Database Locations

  • data/clorag.db - Camera compatibility database
  • data/analytics.db - Search analytics database
  • data/.token_salt - Salt file for OAuth token encryption
Backup Recommendation

Regularly backup the SQLite databases in the data/ directory. These contain camera entries and search analytics that cannot be recovered from vector stores alone.

Local Development

Initial Setup

# Install dependencies
uv sync

# Create .env file from example
cp .env.example .env
# Edit .env with your API keys

Neo4j SSH Tunnel

Production Neo4j is bound to localhost only. For local development:

# Create SSH tunnel to production Neo4j
ssh -L 7687:localhost:7687 root@cyanview.cloud -N -f

# Then set in local .env
NEO4J_URI=bolt://localhost:7687

Local HTTP Development

Set SECURE_COOKIES=false in your local .env to allow cookies over HTTP during development.

Settings

Settings are loaded via clorag.config.get_settings() (cached singleton). Changes to environment variables require server restart.
