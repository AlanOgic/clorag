Odoo MCP Integration

Overview

CLORAG integrates with Odoo ERP via the Odoo MCP Server to provide CRM, sales, and support capabilities directly within the AI assistant. This enables seamless workflows combining knowledge base search with real-time customer data.

What is MCP?

The Model Context Protocol (MCP) is a standard for connecting AI assistants to external tools and data sources. Odoo MCP Server exposes Odoo's API as MCP tools that Claude can invoke.

Key Benefits

  • Unified Support: Search knowledge base AND look up customer data in one conversation
  • Serial Number Tracking: Trace devices through deliveries to customers
  • Warranty Verification: Instant warranty status checks by serial number
  • Sales Integration: Create quotations and view purchase history
  • Repair Management: Create and track RMA/repair orders

Architecture

Communication Flow

  1. CLORAG receives query - Claude agent analyzes intent
  2. Tool selection - Agent chooses Odoo tool (e.g., check_warranty)
  3. OdooMCPClient call - HTTP request to Odoo MCP Server
  4. MCP Server processes - Translates to Odoo v2 API calls
  5. Odoo responds - Data flows back through the chain
  6. Claude synthesizes - Combines with RAG results for response

Data Models

OdooCustomer

OdooCustomer:
  id: int                    # Partner ID
  name: str                  # Display name
  email: str | None          # Email address
  phone: str | None          # Phone number
  mobile: str | None         # Mobile phone
  company_name: str | None   # Parent company name
  country: str | None        # Country name
  country_id: int | None     # Country ID
  vat: str | None            # VAT number
  is_company: bool           # True if company, False if contact

OdooSerial

OdooSerial:
  id: int                    # Lot ID (stock.lot)
  name: str                  # Serial number string
  product_id: int | None     # Product ID
  product_name: str | None   # Product name
  customer_id: int | None    # Customer who received it
  customer_name: str | None  # Customer name
  delivery_date: datetime    # When delivered
  delivery_ref: str | None   # Delivery reference (e.g., WH/OUT/00001)

WarrantyStatus

WarrantyStatus:
  serial: str                # Serial number
  product_name: str | None   # Product name
  product_id: int | None     # Product ID
  purchase_date: datetime    # Original delivery date
  warranty_end: datetime     # Warranty expiration
  is_under_warranty: bool    # Current warranty status
  customer_id: int | None    # Owner customer ID
  customer_name: str | None  # Owner name

ODOO_API_KEY=your-odoo-api-key

Deployment

Docker Compose

The Odoo MCP Server runs as a separate Docker container using the odoo profile:

# Start with Odoo MCP enabled
docker compose --profile odoo up -d

# Start without Odoo (default)
docker compose up -d

# View logs
docker compose logs -f clorag-odoo-mcp

docker-compose.yml excerpt

services:
  clorag-web:
    build: .
    environment:
      - ODOO_MCP_ENABLED=${ODOO_MCP_ENABLED:-false}
      - ODOO_MCP_URL=http://odoo-mcp:8080
      - ODOO_MCP_API_KEY=${ODOO_MCP_API_KEY}

  odoo-mcp:
    image: alanogik/odoo-mcp-19:latest
    environment:
      - ODOO_URL=${ODOO_URL}
      - ODOO_DB=${ODOO_DB}
      - ODOO_USERNAME=${ODOO_USERNAME}
      - ODOO_API_KEY=${ODOO_API_KEY}
      - MCP_TRANSPORT=streamable-http
      - MCP_API_KEY=${ODOO_MCP_API_KEY}
    profiles:
      - odoo  # Only starts with --profile odoo
Platform Compatibility

The Docker image must match the server architecture. Build with docker buildx build --platform linux/amd64 for x86 servers if building on Apple Silicon.

Workflow Examples

Support Workflow

When a support email arrives:

1. lookup_customer(email="customer@example.com")
   → Get customer info and ID

2. get_purchase_history(customer_id=123)
   → See what products they've bought

3. search_serials(customer_id=123)
   → Find all serials delivered to them

4. check_warranty(serial="CY-RIO-48-001")
   → Verify warranty status

5. hybrid_search("RIO connection issue")
   → Search knowledge base

6. create_repair(customer_id=123, product_id=45, description="...")
   → Create RMA if needed

Pre-Sales Workflow

1. search_products(query="RIO")
   → Find relevant products

2. lookup_customer(email="prospect@company.com")
   → Check if customer exists

3. create_customer(name="...", email="...", company="...")
   → Create if not found

4. create_quotation(
     customer_id=456,
     products=[
       {"product_id": 1, "quantity": 2},
       {"product_id": 5, "quantity": 1, "discount": 10}
     ],
     validity_days=30,
     note="Special pricing for new customer"
   )
   → Generate quotation in Odoo

Serial Number Tracing

# Find serial with partial match
search_serials(query="CY-RIO-48")
→ Returns all matching serials with delivery info

# Get detailed info for specific serial
get_serial_info(serial="CY-RIO-48-001")
→ Returns product, customer, delivery date, delivery reference

# List all serials for a product
get_product_serials(product_id=45)
→ Returns all serials with their current owners

Troubleshooting

Common Issues

IssueCauseSolution
Connection refused Odoo MCP Server not running docker compose --profile odoo up -d
401 Unauthorized API key mismatch Verify ODOO_MCP_API_KEY matches on both sides
Timeout errors Slow Odoo server Increase ODOO_MCP_TIMEOUT
Serial not found No stock.lot record Verify serial exists in Odoo Inventory
Platform mismatch arm64 image on amd64 host Rebuild with --platform linux/amd64

Checking Connection

# Check if Odoo MCP Server is running
docker compose --profile odoo ps

# View Odoo MCP Server logs
docker compose logs --tail=50 clorag-odoo-mcp

# Test connection from CLORAG container
docker compose exec clorag-web curl -s http://odoo-mcp:8080/health

Cache Management

The OdooMCPClient uses an LRU cache (100 entries, configurable TTL) for read operations. Cache is cleared on server restart.
