System Architecture

Overview

Cyanview AI Search is a Multi-RAG (Retrieval-Augmented Generation) system that combines multiple knowledge sources for comprehensive technical support:

  • Documentation - Official Docusaurus support documentation
  • Support Cases - Curated and anonymized Gmail support threads
  • Custom Knowledge - Admin-managed documents with full metadata
  • Camera Database - Structured camera compatibility information

Data Flow

The system processes queries through a multi-stage pipeline:

User Query
    ↓
Embeddings Generation (Voyage AI voyage-context-3)
    ↓
Hybrid Search (Dense + Sparse vectors across 3 collections)
    ↓
RRF Fusion (Reciprocal Rank Fusion)
    ↓
Reranking (Voyage AI rerank-2.5 cross-encoder)
    ↓
GraphRAG Enrichment (Neo4j knowledge graph)
    ↓
Claude Sonnet 4.5 Synthesis + Excalidraw Diagrams
    ↓
Streaming Response

Search System

Hybrid Search

The search system uses a hybrid approach combining semantic and keyword matching:

  • Dense vectors (1024-dim) - Voyage AI contextualized embeddings for semantic similarity
  • Sparse vectors - BM25 for exact keyword matching (important for model numbers)
  • RRF Fusion - Reciprocal Rank Fusion combines both results for optimal ranking
  • Reranking - Voyage AI rerank-2.5 cross-encoder refines final relevance (15-40% improvement)

Why Hybrid?

Aspect Dense (Voyage AI) Sparse (BM25)
What It Captures Semantic meaning, concepts Exact keyword matches
Strength "RIO configuration" finds "RCP IP setup" "RIO" finds exactly "RIO"
Best For Conceptual questions, how-to queries Product codes, model numbers, exact terms

Retrieval Settings

  • Retrieved chunks per collection: 10
  • Used for synthesis: 8 top chunks
  • Max chunk size: 2000 characters
  • Chunk overlap: 100 characters
