CLI Commands

All commands are run with uv run <command>. This ensures dependencies are installed and the correct Python environment is used.

Server Commands

uv run rag-web
Start the FastAPI web server on port 8080. Serves the AI Search interface, admin UI, and REST API.
# Start server
uv run rag-web

# Server runs at http://localhost:8080
# Admin at http://localhost:8080/admin
uv run clorag [query]
Run the CLI agent. Without arguments, starts an interactive session. With a query, processes and returns immediately.
# Interactive mode
uv run clorag

# Single query
uv run clorag "How do I configure the RIO IP address?"

Data Ingestion

uv run ingest-gmail [options]
Basic Gmail ingestion without LLM analysis. Fetches and stores raw threads.
Prefer ingest-curated

ingest-curated adds Sonnet classification, quality control, and anonymization. Use ingest-gmail only when you need raw thread data without LLM processing.

uv run ingest-docs [url] [--fresh]
Ingest documentation from a Docusaurus site. Crawls the sitemap, chunks content, and stores in Qdrant.
url
Optional URL override (default: DOCUSAURUS_URL env var)
--fresh
Delete existing collection before re-ingesting (complete refresh)
# Standard ingestion (incremental)
uv run ingest-docs

# Fresh ingestion (delete and rebuild)
uv run ingest-docs --fresh

# Custom URL
uv run ingest-docs https://docs.example.com
uv run ingest-curated [options]
Ingest Gmail support threads with LLM analysis. 7-step pipeline: Fetch → Anonymize → Sonnet classification → Filter → Sonnet QC → Embed → Store.
-m, --max-threads N
Maximum number of threads to fetch
-o, --offset N
Skip first N threads (for incremental ingestion)
-c, --min-confidence N
Minimum confidence for resolved classification (default: 0.7)
--fresh
Delete existing collection before re-ingesting
# Ingest up to 300 threads
uv run ingest-curated --max-threads 300

# Incremental: skip first 300, get next 300
uv run ingest-curated --offset 300 --max-threads 300

# Fresh ingestion
uv run ingest-curated --fresh --max-threads 500
uv run import-docs <folder> [options]
Bulk import custom documents from a folder. Supports .txt, .md, and .pdf files.
folder
Path to folder containing documents
-c, --category
Category: product_info, troubleshooting, configuration, firmware, release_notes, faq, best_practices, pre_sales, internal, other
-t, --tags
Comma-separated tags (e.g., "presales,integration")
-n, --dry-run
Preview documents without importing
# Import presales documents
uv run import-docs ./presales --category pre_sales

# With tags
uv run import-docs ./docs --category product_info --tags "integration,api"

# Preview first
uv run import-docs ./docs --dry-run
uv run ingest-local-docs <path> [options]
Import local markdown documents into Qdrant. Reads .md files from a directory.
path
Path to directory containing .md files
--base-url URL
Base URL prefix for reconstructed page URLs
--fresh
Delete existing collection before importing
--no-cameras
Skip camera extraction step
uv run ingest-legacy-docs [path] [options]
Ingest legacy docs from support.cyanview.com Docusaurus markdown sources into the docusaurus_docs_legacy collection.
path
Path to Docusaurus docs directory (optional)
--site-url URL
Legacy site URL for URL reconstruction
--fresh
Delete legacy collection before re-ingesting
Legacy System

This ingests into a separate collection used by /legacy search. No RIO terminology transforms are applied — original text is preserved for review.

Maintenance Commands

uv run enrich-cameras
Extract camera information from documentation and enrich the camera database. Uses LLM to extract model codes and manufacturer URLs.
uv run enrich-cameras
uv run rebuild-fts [--check]
Rebuild the camera FTS5 (Full-Text Search) index in SQLite.
-c, --check
Only check if FTS index exists and is populated (no rebuild)
# Rebuild index
uv run rebuild-fts

# Check status only
uv run rebuild-fts --check
uv run populate-graph [options]
Populate Neo4j knowledge graph from Qdrant chunks. Extracts entities and relationships using LLM.
--collections COL1 COL2 ...
Qdrant collections to process (default: all 3)
--batch-size N
Chunks per batch (default: 50)
--max-chunks N
Max chunks per collection (default: all)
--concurrency N
Concurrent LLM extractions (default: 5)
# Full population
uv run populate-graph

# Specific collections
uv run populate-graph --collections docusaurus_docs gmail_cases

# Limited for testing
uv run populate-graph --max-chunks 100
uv run fix-rio-terminology [options]
Scan and fix RIO product terminology issues in vector database chunks. Identifies incorrect usage of RIO +WAN, RIO +LAN, etc.
--preview
Scan and save suggestions without applying (default mode)
--apply
Apply all approved fixes to chunks
--stats
Show statistics about terminology fixes
-m, --max-chunks N
Maximum chunks to scan (for testing)
--export FILE
Export fixes to JSON file
--import FILE
Import fixes from JSON file
# Scan for issues (preview)
uv run fix-rio-terminology --preview

# Quick test scan
uv run fix-rio-terminology --preview --max-chunks 20

# Apply approved fixes
uv run fix-rio-terminology --apply

# Check statistics
uv run fix-rio-terminology --stats
Admin UI Available

Review and approve fixes at /admin/terminology-fixes. You can also run scans directly from the UI.

uv run extract-cameras [options]
Extract camera compatibility info from documentation and support cases using Sonnet.
--docs-only
Only extract from documentation chunks
--cases-only
Only extract from support case chunks
--from-sqlite
Extract from support cases SQLite DB
--limit N
Maximum chunks to process
--dry-run
Preview without saving to database
uv run verify-cameras [options]
Verify and correct camera info by looking up manufacturer websites.
--apply
Apply corrections (default is dry-run)
--manufacturer NAME
Only verify cameras from this manufacturer
--limit N
Maximum cameras to verify
--all
Verify all cameras (not just needs_review)
uv run backfill-support-cases
One-time tool to backfill the support cases SQLite database from Qdrant gmail_cases collection.
One-Time Recovery Tool

Use this only when the SQLite support cases DB is empty but Qdrant has data. Not needed for normal operations.

uv run archive-collection [options]
Archive or recover Qdrant collections for backup.

Draft Support

uv run draft-support [options]
Generate draft replies for unanswered support threads. Uses RAG to create contextual responses.
-m, --max-drafts N
Maximum number of drafts to create (default: 10)
-p, --preview
Preview generated responses without creating drafts
-t, --thread ID
Process a specific thread ID
# Create drafts (max 10)
uv run draft-support

# Preview without creating
uv run draft-support --preview

# Process specific thread
uv run draft-support --thread 18f1234567abcdef

# Create up to 5 drafts
uv run draft-support --max-drafts 5
Admin UI Available

Manage drafts at /admin/drafts. Includes preview, manual pipeline runs, and Gmail integration.

Configuration Management

uv run init-prompts [options]
Initialize or manage the LLM prompt database. Creates SQLite tables and populates with default prompts.
--force
Reset all prompts to defaults (auto-backups first)
--list
List all prompts (* = customized)
--category CAT
Filter by category
--stats
Show prompt database statistics
--backup
Backup customized prompts to JSON
-o, --output FILE
Backup to specific path
--restore FILE
Restore customizations from backup
# Initialize defaults
uv run init-prompts

# List all prompts
uv run init-prompts --list

# Backup customizations
uv run init-prompts --backup

# Reset to defaults (auto-backup first)
uv run init-prompts --force

# Restore from backup
uv run init-prompts --restore data/prompt_backups/2025-01-01.json
uv run init-settings [options]
Initialize or manage the RAG settings database. Creates SQLite tables and populates with default settings.
--force
Reset all settings to defaults
--list
List all settings by category
--category CAT
Filter by category
--stats
Show settings database statistics
--export
Export settings to JSON

MCP Server

uv run clorag-mcp
Start the MCP server with stdio transport for Claude Desktop integration. No authentication required.
uv run clorag-mcp-http
Start the MCP server with StreamableHTTP transport on port 8080. Requires MCP_API_KEY for Bearer authentication.
# Local development
MCP_API_KEY=your_key uv run clorag-mcp-http

# Production: https://mcp.cyanview.cloud/ (Docker 8086→8080)

Quality & Development

Development Commands
Linting, type checking, and testing commands for development.
# Install dependencies
uv sync

# Linting (Ruff)
uv run ruff check src/
uv run ruff check src/ --fix  # Auto-fix issues

# Type checking (Mypy)
uv run mypy src/clorag --strict

# Run tests
uv run pytest
uv run pytest tests/test_file.py -v
uv run pytest tests/test_file.py::test_name -v  # Single test

# All quality checks
uv run ruff check src/ && uv run mypy src/clorag --strict
Pro Tip

Most data operations can also be performed from the Admin UI at /admin. The CLI is useful for automation, scripting, and batch operations.
